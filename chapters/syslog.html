NuttX 日志系统（syslog）详解

本文针对 NuttX 深入讲解日志系统子系统——drivers/syslog/ 18 文件 + include/nuttx/syslog/ 3 头子 + libs/libc/syslog/ libc 实现：

  • POSIX syslog 接口：syslog()/vsyslog()/openlog()/closelog()/setlogmask() libc 实现
  • 内核 syslog 接口：syslog_write() / syslog_flush() / syslog_reset() / syslog_bind() / syslog_release() drivers/syslog 主链
  • 多通道分发：console / file / chardev / rpmsg / ram(intbuffer) / early 七类 backend
  • 优先级过滤：LOG_EMERG~LOG_DEBUG 八级 + setlogmask 位掩码
  • 缓冲与刷盘：intbuffer 轮转缓冲、flush 主动刷、reset 重置
  • 启动期 early backend：kmm 未就绪前的早期输出

全部源码引用对应本仓库(nuttx/)里的真实文件，行号即真实行号。

与已写文档的衔面：

  • 与 nuttX_vfs_inode.md 衔——file/chardev backend 走 VFS fd/inode，syslog_bind 设 backend
  • 与 nuttX_irq_management.md 衔——early backend 在 IRQ 上下文可写、console backend ISR safe
  • 与 nuttX_clock_timer.md 衔——syslog timestamp 用 clock_systime / wall time
  • 与 nuttX_memory_mgmt.md 衔——intbuffer kmm_alloc 分配、early backend 静态缓冲避 kmm 未就绪
  • 与 nuttX_taskscheduling.md 衔——syslog_lock 用 sched_lock/IRQ 关临界区、per-thread context
  • 与 nuttX_watchdog.md 衔——panic notifier 常调 syslog_dump 输出最后遗言

1. 概览：syslog 的角色与架构

1.1 syslog 在 NuttX 里干什么

NuttX syslog 是内核与应用共用的日志设施——内核子系统（调度、内存、IRQ、panic）和应用（libc syslog API）走同一套 backend 通道输出日志。区别于 printk 简单串口输出，syslog 支持：

能力 printk syslog
多通道分发✗ 单串口✓ console/file/chardev/rpmsg/ram 七类
优先级过滤✗✓ LOG_EMERG~LOG_DEBUG 八级 + mask
缓冲刷盘✗ 直写✓ intbuffer 缓冲 + flush
POSIX API✗✓ openlog/syslog/closelog/setlogmask
启动期可用✓✓ early backend 静态缓冲
格式化简单✓ timestamp/facility/ident/pid

2. POSIX syslog API：libs/libc/syslog/

2.1 应用接口（include/syslog.h）

API 作用
openlog(ident, option, facility)设应用 ident/option/facility，建会话
syslog(priority, format, ...)按 priority 输出一条日志
vsyslog(priority, format, ap)syslog 的 va_list 版
closelog()关会话
setlogmask(mask)设优先级掩码，返旧值

2.2 priority 八级（include/syslog.h）

宏 数值 语义
LOG_EMERG0系统不可用
LOG_ALERT1必须立即处理
LOG_CRIT2严重条件
LOG_ERR3错误条件
LOG_WARNING4警告条件
NOTICE5普通但重要
LOG_INFO6信息性
LOG_DEBUG7调试级

priority 参数还位或 facility（LOG_KERN/LOG_USER/LOG_MAIL/LOG_DAEMON 等）和 option（LOG_PID/LOG_CONS/LOG_NDELAY/LOG_ODELAY/LOG_NOWAIT）。

2.3 setlogmask 位掩码

setlogmask(mask) 设优先级掩码——mask 是 LOG_MASK(priority) 位或，只输出 priority 位被设的日志。例 setlogmask(LOG_MASK(LOG_ERR)|LOG_MASK(LOG_WARNING)) 只输出 ERR 和 WARNING。

2.4 libc 实现（libs/libc/syslog/）

libc syslog() 实现：

  1. 检查 priority 是否通过当前 mask（否则丢弃）
  2. 格式化消息（vsnprintf 到本地缓冲）
  3. 调内核 syslog_write(priority, buf, len)——libc 是薄封装，实质进 drivers/syslog 主链

3. 内核 syslog 主链：drivers/syslog/

3.1 syslog_write() 主入口（drivers/syslog/syslog_write.c）

void syslog_write(int priority, FAR const void *buffer, size_t buflen)
{
  /* ① 优先级过滤 */
  if ((LOG_MASK(priority) & g_syslog_mask) == 0)
      return;

  /* ② 进 syslog 临界区 */
  syslog_lock();   /* sched_lock + IRQ 关，衔 IRQ 文档 §4 */

  /* ③ 格式化头部：timestamp/facility/ident/pid */
  hdr = syslog_format_header(priority);

  /* ④ 遍历所有活跃 channel 分发 */
  syslog_channel_write(hdr, buffer, buflen);

  /* ⑤ 退临界区 */
  syslog_unlock();
}

关键：syslog_lock 用 sched_lock + IRQ 关——衔 IRQ 文档 §4 临界区机制。syslog 输出是临界区，多 task/ISR 同时写要串行。

3.2 channel 分发：syslog_channel.c

syslog_channel_write() 遍历全局 g_syslog_channel 链——所有活跃 backend 挂在此链，每个收到同一条日志：

void syslog_channel_write(FAR const void *header, FAR const void *body, size_t len)
{
  FAR struct syslog_channel_s *ch;
  list_for_every_entry(&g_syslog_channel, ch, struct syslog_channel_s, node)
    {
      ch->ops->write(ch, header, body, len);   /* 各 backend 实现 */
    }
}

多通道并发：一条日志可同时写串口+文件+RAM 缓冲+rpmsg 转发——这就是 syslog 多通道分发的实质。

3.3 syslog_flush / syslog_reset

API 作用
syslog_flush()主动刷所有 channel（intbuffer 等 flush 到真 backend）
syslog_reset()重置所有 channel（清 intbuffer 等）
syslog_bind()绑指定 channel 为活跃 backend
syslog_release()解绑

4. 七类 backend 详解

4.1 console channel（syslog_console.c + syslog_consolechannel.c）

最常用——输出到串口。up_putstring() arch 钩子直接写 UART，ISR safe（衔 IRQ 文档 §6.4）。启动期默认 console backend。

syslog_console.h 提供 syslog_console_register() 注册 console channel。

4.2 file channel（syslog_filechannel.c）

输出到 VFS 文件——open("/var/log/my.log", O_WRONLY|O_APPEND) 拿 fd，syslog write 调 file_write。衔 VFS 文档 §5。常配 syslog_bind("/var/log/my.log", SYSLOG_FILE)。

4.3 chardev channel（syslog_chardev.c）

输出到字符设备 driver——open("/dev/ttylog0") 走 INODE_IS_DRIVER 路径（VFS 文档 §4.2 分流⑥）。比 file channel 轻，专给日志设备 driver 用。

4.4 dev channel（syslog_devchannel.c + syslog_device.c）

输出到块/MTD 设备——open("/dev/mtd0") 走 INODE_IS_BLOCK/MTD。需 BCH 转字符代理（VFS 文档 §4.2 ③）。用于持久日志存闪存。

4.5 rpmsg channel（syslog_rpmsg.c + syslog_rpmsg_server.c）

SMP 跨核转发——主核 syslog write 经 rpmsg 转到指定核的 backend。CONFIG_FS_RPMSGFS_SERVER 启用。衔 IRQ 文档 §10 SMP。

4.6 intbuffer（syslog_intbuffer.c）

RAM 轮转缓冲——kmm_alloc 一块环形缓冲（CONFIG_SYSLOG_INTBUF_SIZE），syslog write 暂存此处，flush 时刷到真 backend。用途：

  • 启动期真 backend 未就绪时暂存
  • ISR 上下文不能写 file/chardev 时暂存，退出 ISR 后 flush

4.7 early backend（syslog_early.c）

启动期 kmm 未就绪前的早期输出——静态缓冲（非 kmm_alloc），reset 异常时也能用。syslog_initialize()（syslog_initialize.c）启动期先绑 early，kmm 就绪后切真 backend。

5. 启动初始化时序：syslog_initialize.c

void syslog_initialize(void)
{
  /* ① 初始化全局 g_syslog_channel 链 */
  list_initialize(&g_syslog_channel);

  /* ② 注册 early backend（静态缓冲，kmm 未就绪也能用） */
  syslog_early_register();

  /* ③ kmm 就绪后注册 console backend */
#ifdef CONFIG_SYSLOG_CONSOLE
  syslog_console_register();
#endif

  /* ④ 注册其他可选 backend */
#ifdef CONFIG_SYSLOG_FILE
  /* file backend 由 syslog_bind 显式绑 */
#endif
  ...
}

启动期先 early → console，应用调 syslog_bind 切换/增加 file/chardev 等。

8. 总结

  • syslog 是 NuttX 内核与应用共用的日志设施——libc syslog API + 内核 syslog_write 都进 drivers/syslog 主链，支持多通道分发、优先级过滤、缓冲刷盘，远超 printk。
  • 三层抽象：应用 libc/内核 syslog_write → syslog_write.c 主链格式化+过滤 → syslog_channel.c 遍历活跃 backend 分发。
  • 七类 backend：console（串口 ISR safe）/file（VFS fd）/chardev（driver）/dev（块+BCH）/rpmsg（SMP 跨核）/intbuffer（RAM 缓冲）/early（启动期静态缓冲）。syslog_bind 切换/增加。
  • 优先级过滤：LOG_EMERG~LOG_DEBUG 八级 + setlogmask 位掩码，不通过的丢弃。
  • 临界区：syslog_lock 用 sched_lock+IRQ 关，多 task/ISR 同时写串行——衔 IRQ 文档。
  • 启动期 early：kmm 未就绪前用静态缓冲 early backend，kmm 就绪后切真 backend。
  • 衔已写文档面广：VFS（file/chardev backend 走 fd）、IRQ（临界区/ISR safe）、内存（intbuffer kmm）、时钟（timestamp）、看门狗（panic notifier 输出遗言）、SMP（rpmsg 跨核）。
