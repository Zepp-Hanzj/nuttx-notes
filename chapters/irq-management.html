NuttX 中断与 IRQ 管理详解

本文针对 NuttX 深入讲解中断管理的公共机制与 arch 典型实例——IRQ 模型与编号、irq_attach/irq_detach 注册链、irq_dispatch 分发、IRQ 链（irqchain）、enter/leave_critical_section 临界区、IRQ nesting 与优先级、IRQ stack、IRQ 与 sched_lock 的协作、IRQ 退栈前触发 PendSV、信号投递靠 IRQ 路径、SMP 多核 IRQ mask。

全部源码引用对应本仓库（nuttx/）里的真实文件，行号即真实行号。

arch 实例选择：跨 193 文件各 arch 差异大，本文聚焦公共机制（include/nuttx/irq.h + sched/ 公共部分），arch 实例以 Cortex-M NVIC 为典型——Cortex-M 是 NuttX 嵌入式场景主力 arch，其向量表、NVIC 优先级、BASEPRI 临界区、exception return 机制最能代表 RTOS 中断管理全貌。

  • 与任务调度衔接——PendSV/SVC 是软中断、IRQ 退栈前触 PendSV、IRQ 与 sched_lock 协作、IRQ 优先级 vs 任务优先级
  • 与信号衔接——信号投递靠 PendSV 栈框架伪造，IRQ 优先级判定投递时机
  • 与时钟定时器衔接——tick 中断、hrtimer 中断的 IRQ 号与优先级
  • 与看门狗衔接——硬狗超时中断的 IRQ 路径、automonitor BY_CAPTURE 走硬狗中断

  • 与任务创建退出衔接——up_initial_state 伪造现场 LR=task_exit、up_initial_forkstate 双返回根源都是 IRQ exception return 路径
  • 与 VFS inode 衔接——driver open/read/write 跑在 IRQ 上下文下半部

1. 概览：NuttX 中断模型的全景

1.1 三种"中断"来源

NuttX 里"中断"是一个广义概念，包括三种异步事件源：

来源 优先级机制 上下文 典型
硬件外设 IRQNVIC priority（Cortex-M）/ 各 arch 类似IRQ stack 或当前任务栈UART/SPI/timer/GPIO
CPU exception固定优先级（架构定义）exception stackfault、SVC、PendSV、SysTick
软件中断 SWI由 arch 软中断机制定跟 SVC 同SMP 核间调用、up_smp_call

NuttX 把这三种统一抽象为 IRQ 号——irq_attach(irq, isr, arg) 给任意 IRQ 号挂处理函数，arch 负责把硬件 IRQ / exception / SWI 都映射到统一的 IRQ 号空间。

1.2 IRQ 号空间

include/nuttx/irq.h:55-70 定义 IRQ 号空间：

  • NR_IRQS —— arch 提供，总 IRQ 号数
  • Cortex-M 典型：16 exception（0-15）+ 64~240 外设 IRQ（16 起）
  • IRQ 0-15 在 Cortex-M 是 exception（SysTick=15、PendSV=14、SVC=11、reset=1 等）
  • IRQ ≥16 是外设 IRQ，由 NVIC 管理

irq.h:56-70 还定义 IRQ 到压缩索引的映射（IRQ_TO_NDX/NDX_TO_IRQ）——某些 arch IRQ 号稀疏（如 0~1024 但只用了 100 个），用 g_irqmap/g_irqrevmap 把稀疏 IRQ 号压缩成紧凑数组索引，省内存。

2. IRQ 注册：irq_attach / irq_detach

2.1 irq_attach() 公共接口（irq.h:247-280）

int irq_attach(int irq, xcpt_t isr, FAR void *arg);

作用：把 ISR 函数挂到指定 IRQ 号上。挂的是 g_irqmap[irq] 注册表项，不直接管 arch 硬件——up_enable_irq(irq) 才真正打开 NVIC 的对应 IRQ。

参数：

  • irq —— IRQ 号
  • isr —— 处理函数，签名 int (*)(int irq, FAR void *arg, FAR void *regs)（即 xcpt_t）
  • arg —— 透传给 ISR 的参数

返回：OK / -EINVAL（IRQ 越界）/ -ENOMEM（IRQ 链节点分配失败）

2.2 g_irqmap 注册表（irq.h:229-232）

extern irq_mapped_t g_irqmap[];    /* IRQ → ISR+arg 映射表，NR_IRQS 项 */
extern int g_irqrevmap[];          /* 压缩索引 → IRQ 号反向映射（稀疏 IRQ 用） */

irq_mapped_t 是 ISR+arg 联合体（irq.h 邻近定义）。irq_attach 把 isr+arg 填到 g_irqmap[IRQ_TO_NDX(irq)]。

注册表保护：irq_attach / irq_detach 走 irqmap_lock（spinlock）保护 g_irqmap——注册可在启动期单线程跑（无需锁），但运行期 driver 重注册要锁。

2.3 irq_detach() 注销

irq.h 相应接口：把 g_irqmap[irq] 项清空。不调 up_disable_irq——注销只清注册表，是否禁 IRQ 由调用方决定（常配套调 up_disable_irq(irq)）。

2.4 IRQ 链：CONFIG_IRQCHAIN（irq.h:305）

一个 IRQ 号挂多个 ISR 用 IRQ 链——典型场景：GPIO 控制器一个 IRQ 号下挂多个 pin 的 ISR。

int irq_attach_chain(int irq, xcpt_t isr, FAR void *arg);

irq_attach_chain 把新 ISR 节点链到该 IRQ 的链上；ISR 跑时按链顺序调每个节点，节点返回 IRQ_HANDLED 表示已处理（仍继续调后续），IRQ_WAKE_THREAD 表示需唤工作线程（threaded IRQ，CONFIG_IRQPTHREAD）。

IRQ_HANDLED / IRQ_WAKE_THREAD 定义在 irq.h:133,137——IRQ_WAKE_THREAD 用于 threaded IRQ 模式，ISR 跑完后唤 worker thread 跑真正处理（Linux 风格）。

2.5 driver 注册 ISR 的典型模板

/* 启动期把 ISR 挂到 IRQ 号 */
ret = irq_attach(priv->irq, my_isr, dev);
if (ret == OK)
    up_enable_irq(priv->irq);   /* 真正打开 NVIC IRQ */

关键区分：irq_attach 只挂注册表，IRQ 还没开——必须配套 up_enable_irq。注销时反过来：先 up_disable_irq 再 irq_detach，避免 ISR 跑时摘注册。

3. IRQ 分发与 arch 向量表

3.1 arch 向量表（Cortex-M 实例）

Cortex-M 的向量表是一组 void (*)(void) 跳板，每个 IRQ 号一个槽位。arch/arm/cortexm/*/up_vectortable.c 或 linker script 提供。

向量表前 16 项是 exception：

  • [0] 初始 SP
  • [1] reset
  • [11] SVC
  • [14] PendSV
  • [15] SysTick
  • [16+] 外设 IRQ

每个跳板常是汇编 _stubs 或统一 up_doirq 路径——跳到 up_doirq(irq, regs) 调公共 IRQ 处理。

3.2 up_doirq() 分发主链

Cortex-M 的 up_doirq（arch/arm/cortexm/*/up_doirq.c）：

int up_doirq(int irq, FAR void *context)
{
  /* ① 取 g_irqmap[irq] 的 ISR */
  isr = g_irqmap[IRQ_TO_NDX(irq)].isr;
  arg = g_irqmap[IRQ_TO_NDX(irq)].arg;

  /* ② 调 ISR */
  if (isr != NULL)
      isr(irq, arg, context);   /* regs 透传给 ISR */

  /* ③ 触发 PendSV（若有 pending 切换） */
  up_restore_irq(context);   /* exception return，途中可能 bx PendSV */

  return 0;
}

关键：ISR 跑完后 up_restore_irq 走 exception return——若期间有 PendSV pending（sched_unlock merge 出来的），exception return 前会先入 PendSV，做完切换再回被打断的任务。这是"中断退栈前触发 PendSV"机制。

3.3 ISR 签名与 regs 透传

xcpt_t ISR 签名：int (*)(int irq, FAR void *arg, FAR void *regs)。regs 是被打断任务的现场指针——driver ISR 一般不用，但 fault handler 用它做 dump。

3.4 up_enable_irq / up_disable_irq

arch 提供，操作 NVIC：

  • up_enable_irq(irq) —— NVIC ISER 置位，开 IRQ
  • up_disable_irq(irq) —— NVIC ICER 置位，禁 IRQ
  • up_prioritize_irq(irq, priority) —— 设 NVIC IPRIORITYR，priority 是 0~NR_IRQS-1 的 NuttX 优先级（arch 转成 NVIC 优先级位）

3.5 up_irq_save / up_irq_restore

arch 提供，Cortex-M 用 BASEPRI 寄存器：

  • up_irq_save() —— 读当前 BASEPRI，设新 BASEPRI 屏蔽低于某优先级的 IRQ，返回旧值（irqstate_t）
  • up_irq_restore(state) —— 写回旧 BASEPRI

BASEPRI 的语义：设 BASEPRI=N 表示"屏蔽所有优先级 ≥ N 的 IRQ"（Cortex-M 优先级数值越小越高）。这是 Cortex-M 的"软中断禁"——比 PRIMASK 全禁更细，能留高优先级 IRQ 不被禁。NuttX 临界区用 BASEPRI 而非 PRIMASK，留 reset/fault 等最高优先级 exception 不被禁。

4. 临界区：enter_critical_section / leave_critical_section

4.1 公共接口（irq.h:312-355）

irqstate_t enter_critical_section(void);
void leave_critical_section(irqstate_t flags);

作用：进入/退出临界区——屏蔽 IRQ，保护共享数据。NuttX 内核保护共享数据的首选机制，比 spinlock 轻量（无原子操作开销）。

4.2 三种实现模式

irq.h:340-355 显示按编译开关分三种实现：

模式 配置 实现 适用
IRQCOUNT + noinstrumentCONFIG_IRQCOUNTenter_critical_section_notrace() 真函数，thread-specific counterSMP / 需 trace IRQ 嵌套
普通无 IRQCOUNT#define enter_critical_section() up_irq_save() 宏，直调 arch单核典型
notrace无 IRQCOUNT + CONFIG_IRQ_NO_TRACEenter_critical_section_notrace 宏调试期禁 trace

4.3 thread-specific IRQ counter（CONFIG_IRQCOUNT）

SMP 或需 trace 的场景，enter_critical_section 是真函数，用 thread-specific counter：

irqstate_t enter_critical_section_notrace(void)
{
  flags = up_irq_save();
  this_task()->irqcount++;   /* 嵌套计数 */
  return flags;
}

void leave_critical_section(irqstate_t flags)
{
  this_task()->irqcount--;
  if (this_task()->irqcount <= 0)
      up_irq_restore(flags);
}

嵌套支持：临界区可嵌套（一个临界区里调另一个），counter 计数深度，只最外层 leave 时真正 up_irq_restore 开 IRQ。这是 NuttX 临界区可嵌套的根源。

SMP 的意义：SMP 下 up_irq_save 只禁本核 IRQ，跨核共享数据靠 thread-specific counter + spinlock——enter_critical_section 在 SMP 是"本核 IRQ 禁 + 持 spinlock"的组合。

4.4 enter_critical_section vs spinlock

机制 用途 开销 SMP
enter_critical_section短临界区，单核禁 IRQ低（BASEPRI 一条指令）SMP 下配 spinlock
spinlock短临界区，跨核原子操作 + 自旋跨核互斥
mutex长临界区，可阻塞高（持者继承、调度）跨核

NuttX 调度器内部、IRQ 注册表、g_irqmap 等用 enter_critical_section；g_inodelist 等 cross-CPU 共享用 spinlock；driver 长互斥用 mutex。

4.5 irqstate_t

irq.h:175-180 定义 irqstate_t——保存 up_irq_save 返回值，常是 uint8_t（BASEPRI 8 位）。enter_critical_section 返回的 irqstate_t 透给 leave_critical_section 恢复——POSIX sigprocmask 风格的 save/restore 对。

5. IRQ 嵌套与优先级

5.1 Cortex-M NVIC 优先级模型

Cortex-M 的 IRQ 优先级是数值越小优先级越高——与 NuttX 任务优先级（数值越大越高）相反，易混。

NVIC 优先级位数 arch 实现 3~8 位，常 4 位（16 档）。up_prioritize_irq(irq, priority) 把 NuttX 优先级（0~NR_IRQS-1）转 NVIC 数值写 IPRIORITYR。

5.2 IRQ 嵌套

Cortex-M 支持 IRQ 嵌套——高优先级 IRQ 可抢占低优先级 ISR：

  • ISR 跑时默认同优先级 IRQ 不抢占（Cortex-M 硬件规则）
  • 高优先级 IRQ 触发会 nest 当前 ISR——压当前 ISR 现场，切到高优先级 ISR
  • 嵌套深度受栈深度限制

NuttX 配置：CONFIG_ARCH_IRQPRIO 启用 IRQ 优先级管理；CONFIG_ARCH_HAVE_MULTIBLOCK 等控制 NVIC 抢占模型。

flowchart TD Task["任务正常运行"] --> LowIRQ["低优先级 IRQ 触发"] LowIRQ --> LowSave["压任务现场"] LowSave --> LowISR["低优先级 ISR 跑"] LowISR --> HighIRQ["高优先级 IRQ 触发"] HighIRQ --> HighSave["压低优先级 ISR 现场"] HighSave --> HighISR["高优先级 ISR 跑"] HighISR --> HighRet["高优先级 ISR 返回"] HighRet --> HighRestore["恢复低优先级 ISR 现场"] HighRestore --> LowRet["低优先级 ISR 返回"] LowRet --> LowRestore["恢复任务现场"] LowRestore --> TaskBack["回任务继续跑"]

5.3 BASEPRI 控制嵌套

up_irq_save(BASEPRI=N) 屏蔽优先级 ≥ N 的 IRQ——只允许更高优先级 IRQ 嵌套。这是 NuttX 给"部分临界区"的机制：禁大部分 IRQ 但留 reset/fault 等最高优先级 exception 不禁。

5.4 IRQ stack vs 任务栈

Cortex-M exception entry 自动压 R0-R3/R12/LR/PC/xPSR 到当前栈（MSP 或 PSP）——不切专用 IRQ stack。NuttX 任务栈要预留 IRQ 压栈空间（CONFIG_ARCH_INTERRUPTSTACK 或任务栈预留）。

某些 arch（如 Cortex-A）有专用 IRQ stack——exception entry 切到 IRQ 模式栈。Cortex-M 简化在当前栈压。

5.5 CONFIG_ARCH_INTERRUPTSTACK

启用的 arch 给 IRQ 一段专用栈，避免 ISR 跑任务栈导致任务栈不够用。Cortex-M 典型不用（exception 用当前栈），Cortex-A 用。

6. IRQ 与 sched_lock 的协作

6.1 sched_lock 干什么

sched_lock() 增 tcb->lockcount，调度器不切走当前任务——但 IRQ 仍开。用于"我要保护共享数据但不能阻塞，做完一段临界区"。

6.2 IRQ vs sched_lock 的关系

机制 屏蔽对象 适合
enter_critical_sectionIRQ短数据保护，ISR 也跑
sched_lock任务切换长数据保护，IRQ 仍响应

常组合：sched_lock + enter_critical_section——锁调度器防切走，禁 IRQ 防 ISR 乱入。

6.3 IRQ 退栈前触发 PendSV

ISR 跑完要 exception return 回被打断任务。若期间 sched_unlock merge 出 pending 切换（更高优先级任务就绪），exception return 前先入 PendSV 做切换——这是 NuttX "ISR 后切换"的实现：

sequenceDiagram participant Task as 被打断任务 participant HW as 硬件 participant ISR as ISR participant Sched as 调度器 participant Pend as PendSV HW->>HW: IRQ 触发 HW->>HW: exception entry 压现场 HW->>ISR: up_doirq 调 ISR ISR->>Sched: sem_post 唤醒等者 Sched->>Sched: sched_unlock merge
pending 切换 ISR-->>HW: ISR 返回 HW->>HW: up_restore_irq
exception return 前 HW->>Pend: 检测 PendSV pending Pend->>Pend: 上下文切换汇编 Pend->>Sched: 切到新任务 Sched-->>Task: exception return 回原任务

这是"中断退栈前触发 PendSV"机制。NuttX 不在 ISR 里立即切换——ISR 跑完才切，避免 ISR 内做复杂切换逻辑。

6.4 ISR 调 wake 的限制

ISR 跑在 IRQ 上下文，受 ISR 限制：

  • 不能调阻塞 API（sem_wait、read、sleep）
  • 不能持 mutex（mutex 可能阻塞）
  • 不能调 malloc（kmm 多数配置非 IRQ safe）
  • 可以调 sem_post（ISR safe）、sq 入列、设标志

sem_post 从 ISR 唤醒等者——等者加入就绪队列，ISR 退栈时 PendSV 切入。这是 IRQ → 调度器 → 任务的链路。

7. 软中断 SVC / PendSV / SysTick

7.1 SVC

SVC 是同步软中断——任务显式调 SVC #n 触发，常用于内核系统调用入口。NuttX Cortex-M SVC handler 解 SVC #n 的 n 区分调用类型。

7.2 PendSV

PendSV 是最低优先级 exception——故意设最低，让所有 IRQ 都跑完才切。NuttX 用 PendSV 做上下文切换：

  • sched_unlock merge pending → 设 PendSV pending
  • ISR 跑完 exception return 前入 PendSV
  • PendSV handler 做上下文切换汇编

PendSV 最低优先级是 NuttX "ISR 跑完才切" 设计的硬件基础。

7.3 SysTick

SysTick 是 Cortex-M 自带的 tick 定时器——固定优先级（常中等）。NuttX 用 SysTick 做周期 tick（tickless 模式下用 oneshot）。SysTick ISR 调 wd_timer → nxsched_process_tick 等。

7.4 三软中断优先级矩阵

中断 Cortex-M IRQ 号 优先级 用途
SVC11中（常较高）系统调用
PendSV14最低上下文切换
SysTick15中tick

PendSV 最低是关键——保证 ISR 跑完才切。

8. 信号投递的 IRQ 路径

8.1 信号投递靠 PendSV

kill → nxsig_deliver 设 task 的 sigpendingactionq → 设 PendSV pending → ISR 退栈前入 PendSV → PendSV handler 调 nxsig_dispatch 投递信号。

信号的"投递时机"靠 PendSV——PendSV 最低优先级意味着信号投递发生在所有 IRQ 跑完之后，避免在 IRQ 上下文做复杂信号处置。

8.2 投递的栈框架伪造

PendSV handler 调度文档详述的上下文切换汇编，在信号投递时伪造一个栈框架——让任务被切回时先跑信号处置函数，处置完 bx EXC_RETURN 回正常被打断点。这是 IRQ → 信号 → 任务的链路。

8.3 IRQ 优先级判定投递时机

若 IRQ 优先级很高（如 reset、fault），ISR 跑期间 PendSV 不会嵌套（PendSV 最低）——信号投递必须等高优先级 ISR 跑完。这就是 NuttX "信号投递时机靠 IRQ 优先级"的根源。

9. fault 与 exception handler

9.1 Cortex-M fault 类型

Exception IRQ 号 触发
HardFault3严重 fault（栈溢出、非法地址等）
MemManage4MPU 违例
BusFault5总线错误
UsageFault6非法指令、未对齐等
SecureFault7TrustZone

9.2 fault handler

NuttX 的 fault handler（arch/arm/cortexm/*/up_fault.c 或类似）：

  • dump 现场到串口（regs、PC、LR、SP）
  • 调 panic_notifier_chain
  • 进 reset 或 hang

fault handler 跑在 exception 上下文，常不返回——dump 完就 reset 或 hang。

9.3 栈溢出检测

CONFIG_ARCH_STACK_OVERFLOW_ON heap 启用时，任务栈顶 magic 字节被改即触发 HardFault——fault handler 检测到栈溢出。这是 NuttX 栈溢出检测的硬件基础。

10. SMP 多核 IRQ 管理

10.1 per-CPU IRQ mask

SMP 下每核有自己的 IRQ mask——up_irq_save 只禁本核 IRQ，跨核 IRQ 不受影响。这是 SMP IRQ 与单核的关键差异。

10.2 CPU IRQ lock（irq.h:318-322）

CONFIG_SMP 启用 thread-specific IRQ counter 时，enter_critical_section 取 CPU IRQ lock——禁本核 IRQ + 持 CPU 本地锁。跨核共享数据要加 spinlock。

10.3 SWI 核间调用

SMP 跨核派发用 SWI（Software Interrupt）——up_smp_call(cpu, func, arg) 给目标核投 SWI，目标核 SWI ISR 跑 func。这是 SMP 核间通信的 IRQ 路径。

  • 三层抽象：应用调 irq_attach 挂 ISR 到 g_irqmap 注册表；arch 向量表跳 up_doirq 分发；ISR 跑在 IRQ 上下文受 ISR 限制。
  • 临界区：enter_critical_section / leave_critical_section 是 NuttX 内核保护共享数据首选机制，Cortex-M 用 BASEPRI 屏蔽 IRQ（非 PRIMASK，留 reset/fault 不禁）。CONFIG_IRQCOUNT 启用 thread-specific counter 支持嵌套。
  • IRQ 嵌套：Cortex-M 高优先级 IRQ 可嵌套低优先级 ISR，BASEPRI 控制嵌套档位。Cortex-M exception 用当前栈压现场，非专用 IRQ 栈。
  • IRQ 与 sched_lock：常组合——sched_lock 防任务切走，enter_critical_section 防 ISR 乱入。ISR 退栈前触发 PendSV——sched_unlock merge 出 pending 切换在 exception return 前入 PendSV 做切换。
  • PendSV 最低优先级是关键——保证所有 IRQ 跑完才切，避免 ISR 内做复杂切换。信号投递、上下文切换都靠 PendSV。
  • SMP：per-CPU IRQ mask，up_irq_save 只禁本核；跨核共享数据加 spinlock；跨核派发用 SWI 软中断。
  • 衔已写文档面广：PendSV/SVC（调度）、信号投递 IRQ 路径（信号）、tick/hrtimer IRQ（时钟）、硬狗超时 IRQ（看门狗）、ISR 调 sem_post 唤醒（同步）、ISR 调 mq_notify（IPC）、up_initial_state 伪造现场 LR=task_exit（任务创建退出）、driver ISR 跑在 IRQ 上下文（VFS）。
小结一句话：NuttX 中断管理的核心是三层抽象 + 临界区 + PendSV 联动——应用层 irq_attach 挂 ISR 到 g_irqmap 注册表，arch 层向量表 → up_doirq 查表分发，ISR 跑在 IRQ 上下文受限制（不能阻塞/持 mutex/调 malloc，可调 sem_post）。临界区 enter_critical_section 用 BASEPRI 屏蔽 IRQ（留 reset/fault 不禁），CONFIG_IRQCOUNT 支持嵌套计数。Cortex-M 高优先级 IRQ 可嵌套低优先级 ISR。ISR 跑完 up_restore_irq 走 exception return 前检测 PendSV pending——sched_unlock merge 出的切换在 PendSV（最低优先级 exception）里做，保证所有 IRQ 跑完才切。信号投递也靠 PendSV 伪造栈框架。SMP 下 up_irq_save 只禁本核 IRQ，跨核共享加 spinlock，跨核派发用 SWI。
