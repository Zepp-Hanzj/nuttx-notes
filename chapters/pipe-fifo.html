NuttX 管道与 FIFO 详解

本文针对 NuttX 深入讲解管道（pipe）与命名管道（FIFO）子系统——drivers/pipes/ 1390 行三文件（pipe.c / fifo.c / pipe_common.c）：

  • 匿名管道 pipe()：内核内建 inode，两端 fd 共享同一 pipe 实例
  • 命名管道 FIFO：open("/tmp/myfifo") 路径走 driver inode，挂 pipe 实例
  • pipe_common 公共：pipe 实例的环形缓冲、阻塞唤醒、SIGPIPE 投递、f_pos 不复用

全部源码引用对应本仓库(nuttx/)里的真实文件，行号即真实行号。

与已写文档的衔面：

  • 与 nuttX_vfs_inode.md 衔——pipe inode 走 INODE_IS_PIPE 路径（VFS 文档 §4.2），FIFO 走 INODE_IS_DRIVER；fd 挂 fd 表共用 VFS fd 表（VFS 文档 §3.6）
  • 与 nuttX_taskscheduling.md 衔——pipe 阻塞唤醒走 sem_wait/post，TSTATE_BLOCK 状态（调度文档 §3）
  • 与 nuttX_signal.md 衔——写已关端投 SIGPIPE、read 异步通知用 SIGIO
  • 与 nuttX_task_create_exit.md 衔——fork 时 fd 表复制让子继承 pipe fd（fdlist_copy，任务创建退出文档 §4.4）
  • 与 nuttX_irq_management.md 衔——driver ISR 可调 pipe_write 做 IRQ 到任务的缓冲

2. pipe 实例结构：pipe_common.c

2.1 struct pipe_s（drivers/pipes/pipe_common.c 内部，CONFIG_DEV_PIPE_SIZE 配缓冲大小）

struct pipe_s
{
  sem_t   sem;            /* 资源信号量：read 阻塞用，衔 nuttX_semaphore_mutex.md */
  sem_t   semwriter;     /* 写端信号量：write 阻塞用（缓冲满） */
  sem_t   semrdreader;   /* 读端信号量：read 阻塞用（缓冲空） */
  FAR struct pollfd *pollfds;  /* poll/select 等队列 */
  struct pollfd nopoll;        /* poll 空状态 sentinel */
  mutex_t lock;               /* pipe 实例锁 */
  pid_t   pid;                /* 持者（可选） */
  size_t  size;               /* 环形缓冲容量 */
  size_t  nbytes;             /* 当前有效字节 */
  size_t  wrndx;              /* 写索引 */
  size_t  rdndx;              /* 读索引 */
  FAR uint8_t *buffer;        /* 环形缓冲（kmm_alloc） */
  bool    nonblock;           /* O_NONBLOCK 模式 */
  bool    readers;            /* 当前打开的读端数 */
  bool    writers;            /* 当前打开的写端数 */
};

环形缓冲：buffer[wrndx] 写、buffer[rdndx] 读，wrndx/rdndx 在 [0, size) 循环——(wrndx - rdndx) mod size 即有效字节数。nbytes 直接记字节数避免每次算。环形缓冲是 pipe 的核心后端。

三信号量：

  • sem —— 资源信号量，初值 = 缓冲容量，write 调 sem_wait 占一格（满则阻塞），read 调 sem_post 还一格
  • semwriter —— 写端阻塞等（缓冲满）；read 唤醒写端调 sem_post(&semwriter)
  • semrdreader —— 读端阻塞等（缓冲空）；write 唤醒读端调 sem_post(&semrdreader)

三信号量是 pipe 阻塞/唤醒机制的根源——衔 nuttX_semaphore_mutex.md §2 sem_wait/post。

2.2 阻塞 vs 非阻塞

O_NONBLOCK 设 pipe->nonblock = true——read 缓冲空返回 -EAGAIN 而非阻塞，write 缓冲满返回 -EAGAIN。非阻塞模式不调 sem_wait。

2.3 关端计数：readers / writers

pipe->readers / pipe->writers 是当前打开的读/写端数。pipe_open 按 oflags 增对应计数，pipe_close 减。

关键语义：

  • readers == 0（所有读端关了）→ write 返回 -EPIPE 并投 SIGPIPE（衔信号文档）
  • writers == 0（所有写端关了）→ read 返回 0（EOF）——读到缓冲剩字节仍能读出，读完后返 0

3. 匿名管道 pipe()：pipe.c

3.1 pipe() API 链路

int pipe(int fildes[2]) → nx_pipe → pipe_allocate
  1. 分配 pipe 实例：pipe_allocate 建一个 pipe_s，kmm_alloc 缓冲。
  2. 建两个内核内建 inode：一个标记只读（O_RDONLY），一个标记只写（O_WRONLY）——都用 INODE_IS_PIPE 类型，i_ops 是 pipe_read/pipe_write 共用。
  3. 挂到 fd 表：fdlist_dupfile 挂两个 fd 指向两个 file，两个 file 的 f_inode 都指同一个 pipe 实例的 inode（但 oflags 不同）。
  4. 返回：fildes[0] = 读端 fd，fildes[1] = 写端 fd。

关键：pipe 的 inode 不在 VFS 树里——内核内建临时 inode，ref 计数管生命周期。两端 fd 关闭后 inode ref 归零、pipe 实例释放。

3.2 fork 继承 pipe

fork 时 fdlist_copy 复制父 fd 表到子——子也持 pipe fd 的副本，指向同一 file（refs+1）。父关写端子用读端就是经典 pipe IPC 模式。衔任务创建退出文档 §4.4。

3.3 共享同一 pipe_s 实例

虽然两端是两个 inode、两个 file，但两个 inode 的 i_private 都指向同一个 pipe_s 实例——read/write 走公共的 pipe_common 函数操作同一环形缓冲。

4. 命名管道 FIFO：fifo.c

4.1 FIFO 与 pipe 的根本差异

FIFO 是命名的——/tmp/myfifo 是 VFS 树里一个持久的 driver inode，多次 open 同一路径拿到同一 FIFO。pipe 是匿名内核内建 inode，只 fd 持有者能用。

4.2 FIFO inode 注册

register_pipedriver("/tmp/myfifo", &pipe_fops)（drivers/pipes/fifo.c，衔 VFS 文档 §6.4）建一个 INODE_IS_DRIVER inode，i_ops = pipe_fops。这个 inode 持久在 VFS 树里。

4.3 FIFO open 与 pipe 实例生命周期

每次 open("/tmp/myfifo", O_RDWR)：

  1. VFS 走 INODE_IS_DRIVER 路径调 pipe_fops.open（VFS 文档 §4.2 分流⑥）。
  2. 若当前没活跃 pipe 实例：分配新 pipe_s 挂 inode 的 i_private。
  3. 若已有活跃 pipe 实例（前一次 open 没关）：复用同一实例。
  4. 增 pipe->readers/writers 计数。

FIFO 的 pipe 实例生命周期与"有没有人 open"绑定——close 减计数，归零释放实例，但 inode 仍在（持久注册）。下次 open 又建新实例。

4.4 FIFO 阻塞语义

open("/tmp/myfifo", O_RDONLY) 默认阻塞——等到有 O_WRONLY 端 open 才返回。O_NONBLOCK 不阻塞返回。这是 POSIX FIFO 的"读端等写端"语义。

5. read / write 链路：pipe_common.c

5.1 pipe_read 主链路

ssize_t pipe_read(FAR struct file *filep, FAR char *buffer, size_t len)
{
  pipe = filep->f_inode->i_private;

  /* ① 缓冲空 + 有写端：阻塞等 */
  if (pipe->nbytes == 0)
    {
      if (pipe->writers == 0) return 0;   /* EOF 无写端 */
      if (pipe->nonblock) return -EAGAIN;
      sem_wait(&pipe->semrdreader);   /* 等数据 */
    }

  /* ② 从 rdndx 起拷到 buffer，最多 len 或 nbytes */
  nxmutex_lock(&pipe->lock);
  nread = min(len, pipe->nbytes);
  pipe_copy_out(pipe, buffer, nread);   /* 考虑环形 wrap */
  pipe->nbytes -= nread;
  pipe->rdndx  = (pipe->rdndx + nread) % pipe->size;
  nxmutex_unlock(&pipe->lock);

  /* ③ 唤醒写端（腾出空间） */
  sem_post(&pipe->semwriter);

  return nread;
}

5.2 pipe_write 主链路

ssize_t pipe_write(FAR struct file *filep, FAR const char *buffer, size_t len)
{
  pipe = filep->f_inode->i_private;

  /* ① 无读端：投 SIGPIPE 返回 -EPIPE */
  if (pipe->readers == 0)
    {
      nxsig_notification(this_task(), SIGPIPE, SI_KERNEL, ...);  /* 衔信号文档 */
      return -EPIPE;
    }

  /* ② 缓冲满：阻塞等 */
  if (pipe->nbytes == pipe->size)
    {
      if (pipe->nonblock) return -EAGAIN;
      sem_wait(&pipe->semwriter);   /* 等空间 */
    }

  /* ③ 从 wrndx 起拷入 buffer，考虑环形 wrap */
  nxmutex_lock(&pipe->lock);
  nwrite = min(len, pipe->size - pipe->nbytes);
  pipe_copy_in(pipe, buffer, nwrite);
  pipe->nbytes += nwrite;
  pipe->wrndx  = (pipe->wrndx + nwrite) % pipe->size;
  nxmutex_unlock(&pipe->lock);

  /* ④ 唤醒读端 */
  sem_post(&pipe->semrdreader);

  return nwrite;
}

5.3 SIGPIPE 投递（衔信号文档）

写时 readers == 0 → 调 nxsig_notification(this_task(), SIGPIPE, SI_KERNEL, ...) 投 SIGPIPE 给当前任务——衔信号文档 §3。POSIX SIGPIPE 语义根源就在此。

5.4 poll/select 支持

pipe_poll 实现 pollfd 注册——read 端等 semrdreader、write 端等 semwriter，事件就绪时发 SIGIO（若设 O_ASYNC + fcntl F_SETOWN）。

8. 总结

  • pipe 与 FIFO 后端共享 pipe_s 实例——pipe_common.c 实现环形缓冲+三信号量阻塞唤醒，pipe.c / fifo.c 只在建 inode 方式不同。pipe 内建匿名 inode，FIFO 注册持久 driver inode。
  • 环形缓冲：wrndx/rdndx 在 [0,size) 循环，nbytes 计有效字节。三信号量 sem（资源）/semwriter（满阻塞）/semrdreader（空阻塞）是阻塞唤醒根源。
  • 关端语义：readers==0 → write 投 SIGPIPE 返回 -EPIPE；writers==0 → read 返 EOF（0）。衔信号文档。
  • fork 继承：fdlist_copy 让子继承 pipe fd，经典父子 pipe IPC 模式。
  • driver ISR 写入：driver ISR 调 pipe_write 写入做 IRQ 到任务的缓冲，任务 read 出——衔 IRQ 文档。
