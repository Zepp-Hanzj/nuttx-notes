NuttX 内存管理与堆分配器详解

本文针对 NuttX 深入讲解：

  • 用户态堆（umm）与内核态堆（kmm）的双堆架构，及 FLAT/PROTECTED/KERNEL 三种构建模式的差异；
  • 堆分配器（mm_heap）的核心数据结构（allocnode/freenode/nodelist）、malloc/free/realloc 全流程；
  • 内存池（mempool）的定长块分配、ISR 友好的双队列（normal/interrupt）设计；
  • TLSF算法、granular allocation、kmap/iob/shm 等辅助子系统的定位；
  • 与调度器、信号、SMP 的协作，及调试设施（backtrace/kasan/ubsan）。

全部源码引用对应本仓库（nuttx/）里的真实文件，行号即真实行号。

1. 概览：NuttX 内存管理的全景

NuttX 的 mm/ 子系统不是单一分配器，而是多个堆 + 多种分配策略并存的体系。

1.2 三种构建模式下的堆布局

NuttX 有三种构建模式，堆的拓扑完全不同：

2. 核心数据结构：mm_heap_s

所有堆（umm/kmm）本质上都是一个 struct mm_heap_s，定义在 mm/mm_heap/mm.h:221。

2.1 堆主体

struct mm_heap_s
{
  mutex_t    mm_lock;                       /* 互斥锁，控并发访问 */
  size_t     mm_heapsize;                   /* 堆总大小 */
  size_t     mm_maxused;                    /* 历史峰值用 */
  size_t     mm_curused;                    /* 当前已用 */

  FAR struct mm_allocnode_s *mm_heapstart[CONFIG_MM_REGIONS];  /* 各 region 起始 */
  FAR struct mm_allocnode_s *mm_heapend[CONFIG_MM_REGIONS];    /* 各 region 结束 */

#if CONFIG_MM_REGIONS > 1
  int        mm_nregions;                   /* region 数 */
#endif

  struct mm_freenode_s mm_nodelist[MM_NNODES];  /* 二级索引数组，核心！ */

  FAR struct mm_delaynode_s *mm_delaylist[CONFIG_SMP_NCPUS];  /* 延迟 free 链（每核一份） */
  ...
};

关键设计：

  • 多 region 支持 —— 一个堆可挂多段不连续内存（mm_addregion），mm_heapstart/end[] 数组索引；典型用于"SRAM 一段 + SDRAM 一段"拼成一个大堆。
  • mm_nodelist[MM_NNODES] —— 二级索引数组，是性能关键，见 §2.4。
  • mm_delaylist[CONFIG_SMP_NCPUS] —— SMP 下每核一条延迟 free 链，见 §6。

2.2 已分配节点 mm_allocnode_s

mm_heap/mm.h:175：

struct mm_allocnode_s
{
  mmsize_t  preceding;   /* 物理前驱节点大小（用于反向合并） */
  mmsize_t  size;        /* 本节点大小（含头），最低位是 flag:1=已分配 */
#if CONFIG_MM_BACKTRACE >= 0
  pid_t     pid;         /* 调用者 PID（用于 leak 追踪） */
  unsigned long seqno;   /* 分配序号（可选） */
  FAR void *backtrace[CONFIG_MM_BACKTRACE];  /* 调用栈（可选） */
#endif
};

size 字段的最低位是分配标志——size & MM_ALLOC_BIT 为 1 表示已分配。这跟 Linux malloc 的 chunk header 用法同范。preceding 存物理前驱的大小，free 时据此向上合并。

2.3 空闲节点 mm_freenode_s

mm_heap/mm.h:192：

struct mm_freenode_s
{
  mmsize_t  preceding;   /* 同 allocnode */
  mmsize_t  size;        /* 同上，但最低位=0（空闲） */
  /* ... backtrace 字段同 allocnode ... */
  FAR struct mm_freenode_s *flink;  /* 双链前 */
  FAR struct mm_freenode_s *blink;  /* 双链后 */
};

allocnode 和 freenode 复用同一段内存的前缀——chunk 空闲时，头之后的空间被当 freenode 用（挂进 freelist），多了 flink/blink 指针；分配后，这部分变成用户数据区。

2.4 二级索引 mm_nodelist[MM_NNODES] —— 性能核心

MM_NNODES = MM_MAX_SHIFT - MM_MIN_SHIFT + 1（mm.h:116），典型 32 位平台 MM_MIN_SHIFT=4（16字节）、MM_MAX_SHIFT=22（4MB），共 19 档。

mm_nodelist[i] 是一个 freenode 双链表的哨兵头，挂着所有"大小落在 [1<<(i+MM_MIN_SHIFT), 1<<(i+1+MM_MIN_SHIFT)) 区间"的空闲块。即按 size 档位分桶。

档位 0: [16, 32)        字节的空闲块链
档位 1: [32, 64)        字节的空闲块链
档位 2: [64, 128)       字节的空闲块链
...
档位 18: [2Mb, 4Mb]     字节的空闲块链

malloc 时按 size 算出档位 i = LOG2_CEIL(size) - MM_MIN_SHIFT，从 mm_nodelist[i] 起往后扫 first-fit——避免遍历整个 freelist，这是 NuttX 堆分配器的性能关键。

关键源码段（mm_malloc.c line 240-280，精简）：

/* 扫档位找 first-fit */
for (i = minslot; i < MM_NNODES; i++)
{
  for (node = heap->mm_nodelist[i].flink;
       node != &heap->mm_nodelist[i];
       node = node->flink)
  {
    if (node->size >= save)
      goto found;
  }
}

found:
  /* 分裂决策 */
  remaining = node->size - save;
  if (remaining >= MM_MIN_CHUNK + SIZEOF_ALLOCNODE)
  {
    FAR struct mm_freenode_s *newnode =
      (FAR struct mm_freenode_s *)((FAR char *)node + save);
    newnode->size = remaining;
    newnode->preceding = save;
    /* 挂回对应档位 */
    nodelist = &heap->mm_nodelist[LOG2_CEIL(remaining) - MM_MIN_SHIFT];
    newnode->blink = nodelist;
    newnode->flink = nodelist->flink;
    nodelist->flink->blink = newnode;
    nodelist->flink = newnode;
  }
  else
  {
    save = node->size;  /* 整块用掉 */
  }

  node->size = save | MM_ALLOC_BIT;
  heap->mm_curused += save;

3.3 mm_realloc 三策略

mm_realloc.c 按新 size vs 旧 size 分三策略：

策略 触发条件 做什么 开销
①原地扩张newsize > oldsize 且物理后继块空闲且够大分裂后继，合并到当前块最低，无搬移
②搬移①不成立且 newsize > oldsizemm_malloc 新块，memcpy，mm_free 旧块中，有搬移
③收缩newsize < oldsize分裂当前块，后半挂回 freelist低，无搬移

realloc 内部会先尝试①（快路径），失败才②，因此常见场景无搬移开销。

3.5 mm_calloc / mm_zalloc

mm_calloc.c = mm_malloc + memset(0)。mm_zalloc 是内核态版（kmm_calloc）。常用于初始化结构体分配，避免手动清零。

4. 用户态堆 umm 与内核态堆 kmm

4.1 共用 mm_heap，差异在前缀

umm 和 kmm 不是两份独立实现，而是 mm_heap_s 的两份实例 + 薄包装：

项 umm（用户） kmm（内核）
实例g_mmheap 或每进程一份g_kmmheap
入口umm_malloc → mm_malloc(USR_HEAP, …)kmm_malloc → mm_malloc(g_kmmheap, …)
区域用户地址空间内核地址空间
用户接口malloc/calloc/free/realloc（libc）kmalloc/kcalloc/kfree（内核内部）
是否可 ISR 调❌✅（配合 delaylist）

umm_heap/umm_malloc.c 实质就是一行：

FAR void *umm_malloc(size_t size)
{
  return mm_malloc(USR_HEAP, size);
}

kmm_heap/kmm_malloc.c 同范，换 g_kmmheap。

4.2 何时有独立 kmm

仅 CONFIG_MM_KERNEL_HEAP 定义时（PROTECTED/KERNEL 模式），FLAT 模式无独立 kmm，内核与应用共用 g_mmheap。

mm.h:305 kmm_initialize(heap_start, heap_size) —— 内核启动早期调，建 g_kmmheap。

mm.h:300 umm_initialize(heap_start, heap_size) —— 应用启动调，建 g_mmheap。

4.3 多 region 扩展

mm_addregion(heap, start, size)（mm.h:294）给已存在的堆追加一段内存。典型用于：

  • 启动早期只挂了内部 SRAM，后期把外挂 SDRAM 大段也加进来；
  • KERNEL 模式下每进程堆可动态扩 region。

region 间不要求地址连续，mm_heapstart/end[] 数组登记各段；malloc 时若当前 region 不够会试其它。

5. 内存池 mempool

5.1 定位与堆的互补

维度 mm_heap mempool
块大小任意固定 blocksize
分配速度first-fit 扫档位，中等O(1) 取队头
碎片分裂合并，有碎片无碎片，定长
ISR 友好一般（delaylist 兜底）优秀，iqueue 不需锁
扩张固定 region 或 addregionexpand 动态向底层堆要
阻塞等无（失败返 NULL）可选，wait=true 时等 waitsem
典型用应用 malloc、kbmalloc驱动描述符、包缓冲、TCB/inode 节点池

NuttX 内核里大量用 mempool——TCB 节点池、inode 节点池、信号队列节点池、网络 skbuff 等，数量多、大小统一的对象用 mempool 比堆快得多且无碎片。

5.2 数据结构（include/nuttx/mm/mempool.h:92）

struct mempool_s
{
  size_t      blocksize;      /* 每块大小（定长） */
  size_t      initialsize;    /* 普通池初始字节数 */
  size_t      interruptsize;  /* ISR 池初始字节数 */

  bool        wait;           /* 池空时是否阻塞等 */

5.6 池满扩张机制

queue 分配完时，mempool_allocate 调 pool->alloc（mempool_alloc_t 回调）向底层堆要一块 expandsize 字节，切成多个 blocksize 块挂回 queue。这块扩张内存挂在 equeue 上，mempool_deinit 时一并归还底层堆。

扩张是动态的——池子用得多就自动扩，用得少时归还的块自然回 queue（不主动缩）。这跟堆的"固定 region + addregion"不同，mempool 更灵活。

5.7 阻塞等待

wait = true 的池子，任务态取空时阻塞在 waitsem 上（走信号量 wait 路径，task_state = TSTATE_WAIT_SEM）；有块归还时 mempool_release 调 sem_post 唤醒等者。这跟信号量文档 §3 的 sem_wait/sem_post 同构。

6.3 与调度器协作

mm_heap 用 mutex，持锁期间可能 sleep（等锁）；这与任务调度文档 §4.2 的"mutex 不被信号打断"一致。mempool 用 spinlock，持锁期间不禁抢占但禁核间并发，调度器不介入。

ISR 里 mempool_allocate 取 iqueue 不持任何锁、不触切上下文——纯 O(1) 队头操作，ISR 友好的根本所在。

7. 辅助子系统

7.1 mm_gran 颗粒分配（mm/mm_gran/）

位图式固定粒度分配。给定一段内存和粒度 gran_size，用位图每位表示一个粒度块是否占用。

struct gran_s
{
  size_t      ngran;      /* 粒度块数 */
  size_t      granmask;   /* 粒度掩码(gran_size-1) */
  size_t      granshift;  /* 粒度位移(LOG2) */
  FAR void   *heap;       /* 被管内存基址 */
  FAR uint32_t *un;       /* 位图(unused 缩写) */
  mutex_t     lock;       /* 互斥锁 */
};

gran_alloc(gran, size)：

  1. 算需块数 nblk = (size + granmask) >> granshift；
  2. 在位图里找连续 nblk 个 0 位（首次适配）；
  3. 置 1，返回对应内存地址。

gran_free(gran, mem, size)：算出对应位，置 0。

用于 MMU 页表项池、固定粒度的外设寄存器映射这类"大小统一、数量大、不需分裂合并"的场景。比 mempool 更原始——mempool 块可变长（扩张），gran 块完全定粒且按位图管理。

7.2 tlsf（mm/tlsf/mm_tlsf.c）

TLSF（Two-Level Segregated Fit）算法实现。可作为 mempool 的后端（mempool_alloc_t 回调），提供 O(1) 分配 + 低碎片。

TLSF 的二级结构：

  • 第一级：按 2 的幂分桶（类似 mm_heap 的档位）；
  • 第二级：每桶内按线性细分（更精细的子档）。

比 NuttX 默认的"按档位挂 freelist + first-fit"更精细，碎片率更低，但代码量更大。NuttX 默认堆不用 TLSF，但 mempool 配置可选 TLSF 后端。

7.3 kmap / map / shm

子系统 API 作用 实现基础
kmapkmm_map(phys, size, flags)内核态把物理地址映射到内核虚拟地址arch 层 MMU 操作 + mm_heap 兜底

7.4 iob（mm/iob/）

I/O Buffer 子系统，给网络栈、文件系统等用定长缓冲链表（struct iob_s）。

struct iob_s
{
  FAR struct iob_s *next;   /* 链表指针 */
  size_t            io_len; /* 数据长度 */
  size_t            io_offset;
  uint8_t           io_buf[CONFIG_IOB_BUFFER_SIZE]; /* 定长缓冲 */
};

本质是 mempool 的链表变种——多个定长块串成链支持变长数据（网络包可能跨多个 iob），常见于网络 skbuff 风格。iob_alloc 从 mempool 取块，iob_add_entry 链接，iob_free 归还。

11.1 CONFIG_DEBUG_MM 守卫档

mm.h:467 起，#ifdef CONFIG_DEBUG_MM 守卫一整套 ASSERT 宏。开启后：

  • mm_malloc 入口校验 heap != NULL、size > 0、对齐合法性；
  • mm_free 校验 mem 落在 region 内、allocnode->size 最低位==1（已分配，防 double-free）；
  • mm_checkcorruption 遍历时 ASSERT 各 chunk 大小合法、freelist 双链指针互指一致。

关闭时这些 ASSERT 全消失（零开销），适合出厂；开发期必开。

11.2 mm_checkcorruption：堆结构巡检

mm_heap/mm_checkcorruption.c —— 遍历堆所有 chunk（mm_foreach），对每个：

static void checkcorruption_handler(struct mm_allocnode_s *node, void *arg)
{
  size_t nodesize = MM_SIZEOF_NODE(node);

  if (MM_NODE_IS_ALLOC(node))
    {
      ASSERT(nodesize >= MM_SIZEOF_ALLOCNODE);   /* 已分配块至少含头 */
    }
  else  /* 空闲块 */
    {
      FAR struct mm_freenode_s *fnode = (void *)node;
      ASSERT(nodesize >= MM_MIN_CHUNK);          /* 空闲块不小于最小粒 */
      ASSERT(fnode->blink->flink == fnode);       /* 双链互指 */
      ASSERT(fnode->flink == NULL ||
             fnode->flink->blink == fnode);       /* 向后互指 */
      ASSERT(fnode->flink == NULL ||
             MM_SIZEOF_NODE(fnode->flink) == 0 ||
             MM_SIZEOF_NODE(fnode->flink) >= nodesize);  /* 档位有序 */
    }
}

抓的病：

  • 堆踩踏——某块 magic/size 字被踩踏，大小越界或低于最小值，ASSERT 触发；
  • freelist 双链断——blink/flink 不互指，挂接错误；
  • 档位乱序——某档位挂了不属于它大小范围的块，分裂逻辑有 bug。

用法：mm_checkcorruption(heap) —— 常在压测中途或启动时调一次，结构坏即 ASSERT 报错并打 backtrace。

11.3 backtrace + mm_memdump：查内存 leak

11.3.1 每 chunk 存调用栈

mm.h:143 #if CONFIG_MM_BACKTRACE >= 0 时，mm_allocnode_s 多附字段：

struct mm_allocnode_s
{
  ...                              /* preceding/size 同前 */
#if CONFIG_MM_BACKTRACE >= 0
  pid_t                pid;        /* 分配者 PID */
#endif

  unsigned long        seqno;      /* 全局分配序号（可选） */

mm_malloc 分配时调 backtrace API（execinfo.h 的 backtrace()）填 backtrace[]，记 pid = getpid()。mm_free 时这些字段一并失效（块复用为 freenode）。

11.3.2 mm_memdump 输出

mm_heap/mm_memdump.c —— mm_memdump(heap, pid, kind) 按 PID/类型过滤，遍当前所有未释放的 allocnode，输出：

PID 100 seqno 0x1234 allocated 0x20000FF0 size 64
  backtrace: do_xxx+0x10 -> my_alloc+0x20 -> task_main+0x30

特殊 PID 参数（mm_memdump.c line 294-303）：

  • PID_MM_LEAKS —— 输出所有"分配者 PID 已不存在"的块（孤儿 leak，任务死了没释放）；
  • PID_MM_BIGGEST —— 输出最大的 N 块（查"谁吃了最多内存"）；
  • PID_MM_ORPHAN —— 输出所有 backtrace 解析失败的块。

leak 抓法：任务退出时正常应 free 所有分配，nxmq_recover/nxsem_recover 类兜底也回收；若仍有块留着，mm_memdump(heap, 任务PID) 即列出该任务漏释放的块+调用栈，定位 leak 点。

11.4 kasan：UAF/越界/double-free 真运行时检测

11.4.1 影子内存原理

kasan 为每段分配的内存维护影子内存——典型 8 字节主内存对应 1 字节影子。影子值编码：

影子值 含义
0全 8 字节可访问
1..7前 N 字节可访问，后 (8-N) 不可（块内越界）
0xFC (KASAN_GLOBAL_RED)全局红区，未分配
0xFD (KASAN_FREE_RED)已释放，访问即 UAF

kasan.h:83 kasan_poison(addr, size) —— 标"不可访问"；kasan_unpoison(addr, size) —— 标"可访问"。

mm_malloc 调 kasan_unpoison 开用户区；mm_free 调 kasan_poison 全块标红。

11.4.2 编译器插桩 + 报告

mm/kasan/hook.c —— 实现 GCC/Clang 的 ASan runtime 接口（__asan_loadN/__asan_storeN 等）。编译器开启 -fsanitize=address 后，每次读写内存前插桩调这些函数，函数查影子值：

void __asan_load8(void *addr)
{
  kasan_check_report(addr, 8, false, return_address(0));
  /* 影子标红 → kasan_report 打印+PANIC */
}

kasan_report（hook.c）打印越界类型、地址、调用栈，触发 PANIC（可配 CONFIG_MM_KASAN_DISABLE_READ_PANIC 不 panic 只记）。

11.4.3 抓的四类病

病 怎么抓 影子状态
UAF（use-after-free）free 后块标 0xFD，后续访问插桩查影子即红0xFD

double-freemm_free 第一次后块标 0xFD，二次 free 时 ASSERT size 最低位==1 失败 + 影子红0xFD + ASSERT
leakmm_memdump 配 kasan 影子状态可查"标红但仍被引用"的块0xFD

小结一句话：NuttX 内存管理 = 双堆（umm 用户 / kmm 内核）+ 二级索引 first-fit 堆分配器 + mempool ISR 友好定长池 + gran 位图 + iob 链式缓冲。堆靠 mm_nodelist[] 档位索引加速 first-fit，靠 preceding 链式合并防碎；mempool 靠 queue/iqueue 双队列让 ISR 无锁取块；SMP 下每核独立 mm_delaylist 兜 ISR 里抢不到锁的 free；三种构建模式（FLAT/PROTECTED/KERNEL）通过 USR_HEAP/KRN_HEAP 宏抽象堆拓扑差异。调试上 backtrace 存 PID/调用栈、kasan 影子内存、checkcorruption 校 magic，三套互补防漏防越。
