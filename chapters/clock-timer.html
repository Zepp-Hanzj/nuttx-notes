1. 概览：NuttX 时钟子系统的全景

摘链——若摘的是队头，重设 SysTick 让下次中断对新队头。回调不跑（取消时机未到）。

4.4 超时回调的用途

wdog 回调不在目标任务上下文跑，在 tick ISR / hrtimer 处理里跑——因此不能持 mutex、不能阻塞，只能做"摘队列 + 补 errcode + 唤醒"：

nxmq_sndtimeout / rcvtimeoutmq_sndinternal.c / mq_rcvinternal.cnxmq_wait_irq(tcb, ETIMEDOUT)

用户 wdog应用调 wd_start用户回调（仍不能阻塞）

5. hrtimer 高精度定时器

5.1 数据结构（include/nuttx/hrtimer.h:93）

typedef struct hrtimer_s
{
  hrtimer_node_t  node;       /* RB-Tree 节点或 list 节点 */
  hrtimer_entry_t func;       /* 到期回调 */
  uint64_t     expired;       /* 绝对到期纳秒 */
} hrtimer_t;

typedef CODE uint64_t (*hrtimer_entry_t)(const hrtimer_t *hrtimer,
                                          uint64_t expired);

回调返 uint64_t——非0 表示重新启动的相对延迟（实现周期定时器），0 表示一次性。

5.2 组织：RB-Tree vs 链表

CONFIG_HRTIMER_TREE 时用 RB-Tree（hrtimer_node_t = RB_ENTRY），否则单链表。RB-Tree 按 expired 排序，最左节点是最早到期——hrtimer_process 取最左处理。RB-Tree 适合大量定时器（O(log n)），链表适合少量（O(n) 扫）。

6. tickless 算法详解

6.1 为什么要 tickless

非 tickless 嵌入式设备闲时 SysTick 仍周期中断（每 10ms），CPU 反复唤醒——空耗功耗。tickless 让闲时 SysTick 设为"下次有事的时间"，无事则停（设最大计数），功耗骤降。

6.2 "下次有事"的三个来源

up_tickless() 算下次中断时间：

6.3 与空闲任务的协作

tickless 设 SYSTICK_MAXCOUNT 后，CPU 进空闲任务（sched_idletask）做低功耗态（up_idle() 架构层 WFI/WFE）。下次有事时 SysTick 中断唤醒 CPU，或外设 IRQ 唤醒——这是 tickless 的核心省功耗路径。

6.4 arch 层实现差异

各 SoC 的 tickless 实现不一：

  • arch/arm/src/armv7-m/up_tickless.c —— Cortex-M 通用 SysTick 32 位计数；
  • arch/risc-v/src/common/espressif/esp_tickless.c —— ESP32 系专用，处理 ESP32 的 RTC + SysTick 组合；
  • 板级 tickless.h —— 如 esp32c3_tickless.h 配具体时钟源。

差异在"用哪个硬件计数器、精度多少、低功耗态怎么进"，核心算法同 §6.2。

7. POSIX timer（应用层）

timer_create/timer_settime/timer_gettime/timer_delete —— POSIX 应用层定时器，建在 hrtimer 上：

struct timer_s
{
  struct sigevent event;   /* 到期发什么信号 */
  hrtimer_t       hrtimer; /* 底层 hrtimer */
  ...
};

timer_settime 调 hrtimer_start，到期回调 nxsig_notification 给目标进程发信号（走信号文档 §3 投递流程）。这是应用用 hrtimer 的主接口——比 wdog 友好（wdog 是内核态、回调不能阻塞）。

9. 实战：sem_timedwait 超时回收全时序

设场景：任务 A 调 sem_timedwait(sem, {1s})，1 秒内未拿到信号量。

sequenceDiagram participant A as 任务 A participant Sem as sem->waitlist participant Wdog as wdog 看门狗 participant Tick as tick 中断 / hrtimer participant Sched as 调度器 A->>Sem: T0: sem_timedwait(sem, timeout=1s) A->>Sem: T1: nxsem_wait_slow 阻塞, 入 sem->waitlist A->>Wdog: T2: wd_start(wdog, 1s, nxsem_rcvtimeout, sem) Note over A: T3: A 阻塞 (TSTATE_WAIT_SEM) Note over Tick: T4: 1 秒到 (tick 中断或 hrtimer 到期) Tick->>Wdog: T5: wd_timer/wd_timer_callback 摘到期 wdog Tick->>Tick: T6: 调 nxsem_rcvtimeout(sem) 超时回调 Tick->>Sem: T7: nxsem_wait_irq(A, ETIMEDOUT) Sem->>Sem: T8: 摘 A 离 sem->waitlist Sem->>A: T9: A->errcode = ETIMEDOUT Sem->>Sched: T10: nxsched_add_readytorun(A) 唤醒 Sched->>Sched: T11: 若队头变 → up_switch_context svc 切 A->>A: T12: A 醒, sem_timedwait 返 -ETIMEDOUT

关键：超时回调（T6）在 tick ISR / hrtimer 处理上下文跑，不持 mutex、不阻塞——只摘队列补 errcode 唤醒，真切换延后到 svc/PendSV（对应任务调度 §5）。

小结一句话：NuttX 时钟 = 三层并存（tick 级 clock 走系统时间+调度轮转+CPULoad，纳秒级 hrtimer 走精定时+RB-Tree，wdog 走超时看门狗可建在 tick 或 hrtimer 上）。tick 模式周期中断，tickless 模式重设 SysTick 为"下次有事"（就绪/wdog/hrtimer 三源最早）省功耗闲时 CPU 进 idle。wdog 超时回调在 ISR 跑不能阻塞，常补 IPC 等者 ETIMEDOUT；hrtimer 回调同样约束但支持周期返非0 重启。sem_timedwait/mq_timed/sigtimedwait/nanosleep/POSIX timer 全走 wdog/hrtimer，超时回收走 nxsem/nxmq_wait_irq/nxsig_wait_irq 同构路径，真切换延后到 svc/PendSV。
